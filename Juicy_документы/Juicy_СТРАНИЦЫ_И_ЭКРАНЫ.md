> **Назначение.** Полный список всех экранов прототипа «как есть» (по коду `app/` на момент сборки)

> админку — desktop ≥1280px. Если у экрана два варианта (авторизован/нет) — два файла с суффиксами
> `-auth` / `-guest`.

## Часть 1. ПУБЛИЧНЫЙ САЙТ (клиентское mobile-web приложение)

### 1.0 Что значит «авторизован»

Авторизация в прототипе — это наличие `user.phone` в Zustand-store (localStorage). Появляется после
прохождения цепочки **A4 Телефон → A5 OTP → A6 Имя**. Кнопка «Выйти» в профиле стирает телефон.

**Сводная таблица различий авторизован / гость:**

| Экран | Гость (не авторизован) | Авторизованный |
|---|---|---|
| A1 Splash, A2 Онбординг, A3 Точки | одинаково | одинаково |
| A4–A6 Auth-флоу | доступен, это путь входа | штатно не посещает (но маршрут не закрыт) |
| B1 Главная, B2 Меню, C1 Продукт | **одинаково** (просмотр и корзина доступны без входа) | одинаково |
| D1 Корзина | UI идентичен, но кнопка **«Оформить» ведёт на `/auth/phone`** | кнопка «Оформить» ведёт на `/checkout` |
| D2 Checkout | штатно недоступен (флоу заворачивает на вход); прямой заход покажет пустой блок «Контакт» | блок «Контакт» показывает телефон с галочкой «подтверждён» |
| D3 Оплата | штатно недоступен | доступен |
| E1 Заказы | одинаковый UI; у гостя всегда пустое состояние | список заказов из localStorage |
| E2 Статус заказа | недостижим (нет заказов) | полный экран статуса |
| F1 Профиль | **отдельный гостевой экран**: 👋 «Войди в Juicy» + кнопка «Войти по телефону» | полный профиль (аватар, статистика, машина, язык, помощь, выйти) |

> ⚠️ Жёстких route-guard'ов в прототипе нет: проверка только в кнопке корзины и в рендере профиля.
> На бэке (BACKEND.md) `/orders`, `/checkout`, `/payment` потребуют JWT.

### AD1. Дашборд — `/admin`

> 📷 `screens/admin/ad1-dashboard.png`

- **4 стат-карточки:** «Заказов сегодня» (+12% к вчерашнему) · «Выручка, AED» (+8.4%) · «Среднее время готовки 3:24» (−18 сек) · «Точек онлайн N/M» («City Walk закрыт»).

### AD3. Деталка заказа — `/admin/orders/[id]`

> 📷 `screens/admin/ad3-order-detail.png`, `ad3-order-notify.png`, `ad3-order-cancel-confirm.png`

- **Действия в шапке:** «Печать чека» (тост «отправлен на термопринтер») · **«Отменить»** (красная; disabled для CANCELLED/PICKED_UP; диалог подтверждения с предупреждением о Refund).
- **Левая колонка:** панель **«Состав заказа»** (+ статус-бейдж): таблица Позиция (название + вариант) / Допы (пилюли) / Кол-во / Сумма + строка «Итого» · панели **«Клиент»** (имя, телефон, «Машина для curbside» крупным моно) и **«Оплата»** (метод, Tap charge id, точка).
- **Правая колонка:**

  - **«Оповестить клиента»** («SSE + Web Push (V2) + SMS-fallback (опц.)»): кнопки **«Готов — забирайте» / «Задержка» / «Отмена» / «Своё сообщение»** (textarea + Отправить) · **журнал отправок** (template · время · канал);
  - **«Хронология»**: таймлайн статусов с временем и автором («автомат» / имя).

### AD4. Блюда — `/admin/catalog/products`

- **Фильтры:** поиск по названию · селект категории · «Показано N из M».
- **Таблица:** превью / Название (ссылка в редактор; + бейдж) / Категория / Варианты (число) / Группы допов (число) / Цена от (AED) / КБЖУ (ккал) / Медиа («video + poster» | «только poster») / Статус («активно» | «стоп» | «скрыто») / «Открыть».
- **Пустое состояние:** «Ничего не найдено по фильтрам» + «Сбросить».

### AD5. Редактор блюда — `/admin/catalog/products/[slug]` (6 вкладок)

> 📷 `screens/admin/ad5-product-main.png`, `-media.png`, `-variants.png`, `-bindings.png`, `-rules.png`, `-outlets.png`, `ad5-product-sandbox.png`

- **Шапка:** slug + категория + бейдж · тогглы **«Видимость»** и **«В наличии»** · кнопки «Дублировать» / «Отмена» / **«Сохранить»** (с «●» при несохранённых изменениях).
- **Вкладка «Основное»:** панель «Локализованные тексты»: Название RU/EN, Описание RU, Категория, Бейдж (BESTSELLER/NEW/LIMITED/SEASONAL), Время готовки (сек), Теги · панель **«КБЖУ базы (размер M)»** («с допами пересчитывается»): 4 ячейки Ккал/Белки/Жиры/Углев. + Сахар, Клетчатка, Вес/объём базы.
- **Вкладка «Медиа»:** «Видео-петля» (mp4 720×1280 ≤800 KB; путь cdn.juicy.ae/products/{slug}.mp4; «Заменить»/«Выбрать файл») · «Превью (постер первого кадра)» (обязательно; «Автогенерация из видео» / «Загрузить вручную») · «Прозрачный PNG для каталога» (опционально) + «Цвет фона».
- **Вкладка «Варианты»:** таблица размеров S/M/L: Код / Название / Объём мл / Цена AED / ΔКкал / ΔБелки / ΔЖиры / ΔУглев. / В наличии (тоггл) / удалить · «+ Добавить вариант».
- **Вкладка «Допы» (bindings):** панель **«Привязанные группы допов»**: Группа / Тип выбора (одно·несколько·счётчик) / Required (тоггл) / Min / Max / Default / отвязать; «+ Группу» → модал выбора · панель **«Цена и доза допов в этом блюде»** (override): Доп (с базовыми ценой и дозой) / Цена за unit / Доза в блюде / Min units / Max units / Скрыть; зелёная подсветка = перекрывает базу; «+ Override» → модал.
- **Вкладка «Правила»:** правила уровня блюда: «Когда» → «Эффект» → заметка + тоггл + удалить («овсяное молоко → цена овсяного = 0 AED»; «decaf шот → запрет: обычный шот») · «+ Правило» · блок **«Песочница "что будет, если…"»** → модал: чипы допов + превью ответа `POST /products/{slug}/preview` (allowedAddons / forbiddenAddons / priceAed / nutrition).
- **Вкладка «Точки»:** таблица per-outlet: Точка / В меню (тоггл) / В наличии сейчас (тоггл) / Стоп-лист с (время · кто) / «Журнал».

### AD6. Допы — `/admin/catalog/addons`

> 📷 `screens/admin/ad6-addons.png`, `ad6-addon-drawer.png`, `ad6-addon-new-modal.png`

- **Действие:** **«+ Новый доп»** → модал: Название (RU) + Группа.

Изменение цены повлияет на новые заказы — снэпшоты в БД останутся прежними» · «Удалить» (подтверждение: «Заказы, где он использовался, сохранятся: цена и КБЖУ снэпшотятся») / «Отмена» / «Сохранить».

### AD7. Группы и правила — `/admin/catalog/groups` (2 вкладки)

> 📷 `screens/admin/ad7-groups.png`, `ad7-rules.png`, `ad7-group-composition.png`, `ad7-rules-sandbox.png`

- **Вкладка «Группы допов»:** пояснение «Группа фиксирует тип выбора… required/min/max — в самом блюде» · таблица: Группа / Slug / **Тип выбора** (переключатель один·несколько·счётчик прямо в строке) / Допов в группе / Используется в блюдах / Видимость / **«Состав»** → модал со списком допов группы (доза·цена + тоггл) · «+ Новая группа» → модал (название + тип выбора).
- **Вкладка «Правила комбинирования»:** пояснение «срабатывает, когда выбраны все аддоны из "Когда"… global/product» · таблица: # / Когда выбрано (пилюли) / → Эффект (**запрет** | **скидка N%**) / Scope («все блюда» | slug) / Сообщение клиенту / Активно (тоггл) / Редактировать·удалить · «+ Новое правило» (создаёт выключенный черновик) · панель **«Песочница правил (dry-run)»** → модал: блюдо + выбранные допы + JSON-превью ответа preview.

> 📷 `screens/admin/ad10-staff.png`, `ad10-staff-drawer.png`, `ad10-staff-invite-modal.png`

- **Таблица «Все сотрудники»** (7 учёток в моке): аватар-инициалы + Имя / Email / Роли (пилюли) / Доступ к точкам («все точки» | список) / **2FA** («требуется» для super_admin и finance) / Последний вход / Статус (активен·отключён) / «Изменить».
- **Drawer редактирования:** Имя · Email · **Роли (мультивыбор чекбоксами, все 6)** · Доступ к точкам: тоггл «Все точки сети» или чекбоксы по точкам · тоггл «Активная учётка / Учётка отключена» · «Аудит: 17 действий за 7 дней» · Удалить (подтверждение «история действий и заказов останется») / Сохранить.
