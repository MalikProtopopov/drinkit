# GRABZI frontend parser

Парсер фронтенда магазина [grabzi.ae](https://grabzi.ae/) (платформа **Shopify**).

## Что забирает

| Данные | Источник |
|---|---|
| Каталог товаров (цены, варианты, изображения, описания) | Shopify JSON API `/products.json` (с пагинацией) |
| Коллекции | `/collections.json` |
| Навигация / меню | ссылки `/pages/`, `/collections/`, `/products/` с главной |
| Контентные страницы (`menu`, `story`, `contact-us`, `working-hours`) | HTML `/pages/*` |
| Секции главной (hero, заголовки, картинки) | разбор HTML через BeautifulSoup |
| Мета сайта (title, og:*, шрифты) | `<head>` главной |

## Установка

```bash
pip install -r requirements.txt
```

## Запуск

```bash
python3 grabzi_parser.py            # спарсить всё в ./output
python3 grabzi_parser.py --images   # дополнительно скачать изображения товаров
python3 grabzi_parser.py --base https://grabzi.ae/ --out ./output
```

## Результат (каталог `output/`)

- `products.json` — нормализованный каталог товаров
- `pages.json` — контентные страницы (заголовок, текст, ссылки)
- `site.json` — всё вместе (мета, навигация, коллекции, товары, страницы, секции)
- `summary.md` — читаемая сводка
- `images/` — изображения товаров (только с флагом `--images`)

## Заметки

- Сайт на Shopify, поэтому каталог берётся из официального JSON-эндпоинта —
  это надёжнее, чем парсить отрендеренный HTML. HTML разбирается только для
  контента, которого нет в API (страницы, секции главной, навигация).
- Между запросами выдержана пауза (`POLITE_DELAY`) и стоят ретраи, чтобы не
  нагружать сервер и пережить временные сбои сети.
